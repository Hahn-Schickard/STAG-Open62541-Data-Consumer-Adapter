391 /* Use the type versions of this method. See below. However, the general
392  * mechanism of async service calls is explained here.
393  *
394  * We say that an async service call has been dispatched once this method
395  * returns UA_STATUSCODE_GOOD. If there is an error after an async service has
396  * been dispatched, the callback is called with an "empty" response where the
397  * statusCode has been set accordingly. This is also done if the client is
398  * shutting down and the list of dispatched async services is emptied.
399  *
400  * The statusCode received when the client is shutting down is
401  * UA_STATUSCODE_BADSHUTDOWN.
402  *
403  * The statusCode received when the client don't receive response
404  * after specified config->timeout (in ms) is
405  * UA_STATUSCODE_BADTIMEOUT.
406  *
407  * Instead, you can use __UA_Client_AsyncServiceEx to specify
408  * a custom timeout
409  *
410  * The userdata and requestId arguments can be NULL. */
