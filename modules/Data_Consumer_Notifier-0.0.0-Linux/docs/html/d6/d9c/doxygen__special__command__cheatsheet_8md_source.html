1 # Doxygen special command cheatsheet
2 
3 This list is based on [star.bnl.gov](https://www.star.bnl.gov/public/comp/sofi/doxygen/commands.html#cmdauthor) list. It has been curated down to most relevant commands for code documentation. For full list please visit the previously mention website or [doxygen.nl](http://www.doxygen.nl/manual/commands.html) for more information.
4 
5 All commands in the documentation start with a backslash (\) or an at-sign (@). If you prefer you can replace all commands starting with a backslash below, by their counterparts that start with an at-sign.
6 
7 Some commands have one or more arguments. Each argument has a certain range:
8 
9  If <sharp> braces are used the argument is a single word.
10  If (round) braces are used the argument extends until the end of the line on which the command was found.
11  If {curly} braces are used the argument extends until the next paragraph. Paragraphs are delimited by a blank line or by a section indicator.
12 
13 If [square] brackets are used the argument is optional.
14 
15 ## Command list
16 
17  \@a <word> Displays the argument <word> in italics. <br>
18  \@arg { item-description } Describes a simple not nested argument. <br>
19  \@attention { text } Starts a paragraph where a message that needs attention may be entered. <br>
20  \@author { list of authors } Starts a paragraph where one or more author names may be entered. <br>
21  \@b <word> Displays the argument <word> using a bold font. <br>
22  \@brief { text } Starts a paragraph that serves as a brief description. <br>
23  \@bug { bug description } Starts a paragraph where one or more bugs may be reported. <br>
24  \@c <word> Displays the argument <word> using a typewriter font. <br>
25  \@code [ '{'<word>'}'] Starts a code block. Everything in this block is treated as source code, and proper highlights will be used to showthe-content. losed of with \endcode <br>
26  \@date { date description } Starts a paragraph where one or more dates may be entered. <br>
27  \@deprecated { description } Starts a paragraph indicating that this documentation block belongs to a deprecated entity. <br>
28  \@e <word> Displays the argument <word> in italics. <br>
29  \@exception <exc> { text } Starts an exception description for an exception object with name <exc> <br>
30  \@f$ <formulae> \f$ Marks an inline formulae. <br>
31  \@f[ <formulae> \f] Marks a long center lined formulae. <br>
32  \@invariant { text } Starts a paragraph where the invariant of an entity can be described. <br>
33  \@li { item-description } Marks a single element of an unordered list. <br>
34  \@note { text } Starts a paragraph where a note can be entered. <br>
35  \@param <par> { text } Starts a parameter description for a function parameter with name <par>. <br>
36  \@post { text } Starts a paragraph where the postcondition of an entity can be described. <br>
37  \@pre { text } Starts a paragraph where the precondition of an entity can be described. <br>
38  \@remarks { text } Starts a paragraph where one or more remarks may be entered. <br>
39  \@return { text } Starts a return value description for a function. <br>
40  \@retval <value> { text } Starts a paragraph where return type with name <value> can be entered. <br>
41  \@since { text } This tag can be used to specify since when (version or time) an entity is available. <br>
42  \@test { text } Starts a paragraph where a test case can be described. <br>
43  \@todo { text } Starts a paragraph, where notes for future work may be entered. <br>
44  \@version { version number } Starts a paragraph where one or more version strings may be entered. <br>
45  \@warning { text } Starts a paragraph where one or more warning messages may be entered. <br>
46  \@~[LanguageId] Specifies language translation options. <br>
